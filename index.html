O que posso fazer para você?

Teste Manual

Geralmente é utilizado para identificar problemas de usabilidade, falhas no código-fonte ou em situações que envolvem o pensamento lógico. Além disso, o teste manual é adotado para sistemas com um curto ciclo de vida.

Teste Automatizado

O teste automatizado de software é aquele que utiliza uma ferramenta de TI específica para simular o uso da aplicação / sistema. Isso pode ser feito também com o apoio de scripts. Esse processo tem como maior vantagem a agilidade. Inúmeros inputs podem ser executados rapidamente, garantindo que o software atenda as funcionalidades especificadas nos requisitos.

Elaboração de roteiro de teste e relatório de defeitos

Elaborado a partir dos documentos de especificação de um determinado caso de uso, como: especificação funcional, guia de interface e modelagem do banco de dados. O roteiro de teste também é conhecido como Projeto de Teste, Script de Teste ou Especificação de Teste.
